# Build Log 자동 생성 프롬프트 가이드

## 목적

커밋 데이터를 기반으로, 단순 커밋 나열이 아닌 **AI 활용 교육 콘텐츠**를 자동 생성한다.

## 핵심 원칙

1. **커밋은 맥락일 뿐** — 커밋 목록을 나열하지 않는다
2. **방법론이 본문** — 프롬프팅 기법, 구조화 전략, 도구 활용법이 핵심
3. **교육적 가치** — 독자가 읽고 바로 써먹을 수 있어야 한다
4. **팩트 기반 개선안** — 공식 문서, 최신 릴리스 기반으로 더 나은 방법 제안

## 포스트 구조

```
1. 도입부 (h2 없이, 2~3줄)
   → 무엇을 했고, 이 글에서 뭘 배울 수 있는지

2. 배경: 무엇을 만들고 있는가 (h2)
   → 프로젝트 소개, 이번 작업 목표

3. 핵심 섹션 2~3개 (h2) — 각 섹션이 하나의 "레슨"
   a) 프롬프팅 전략 — 좋은 프롬프트 vs 나쁜 프롬프트
   b) Claude Code / AI 도구 활용법 — CLAUDE.md, hooks, MCP, skills
   c) 구조화 전략 — 작업 분할, 제약 조건 설정, 자동화 패턴
   d) 관련 기술 개념 설명

4. 더 나은 방법은 없을까 (h2)
   → 공식 문서 best practice, 최신 기능, 커뮤니티 패턴
   → 다른 도구/MCP 서버 대안
   → 성능/비용/품질 트레이드오프

5. 정리 (h2)
   → 핵심 포인트 3~4개, 각 한 줄

6. 참고한 커밋들 (접기)
   → <details> 태그로 접어서 하단에
```

## 문체

- 건조한 반말 (toss tech 톤)
- 기술 용어 번역 안 함
- 감탄사/이모지 금지
- 인라인 코드는 backtick

## 제목 규칙

**방법론/인사이트**가 드러나야 한다.

좋은 예:
- "Claude Code로 i18n 8개 언어를 한번에 처리하는 프롬프트 패턴"
- "AI 에이전트에게 리팩토링 시키는 법 — 제약 조건이 핵심이다"
- "CLAUDE.md 하나로 코드 품질 올리는 구조화 전략"

나쁜 예:
- "작업 정리", "업데이트", "[프로젝트명] 외 N건"

## 분량

최소 3,000자 이상 (공백 포함)
