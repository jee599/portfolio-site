25시간짜리 세션이 있었다. `claude-opus-4-6`, 261 tool calls. 목표는 하나였다 — 네이버 치과 블로그에 들어갈 이미지 24장을 자동 생성하는 파이프라인을 만드는 것. 결과는 `pipeline.py` + Gemini API + HTML 템플릿 7종으로 구성된 시스템이었다.

**TL;DR**: 이미지 생성 파이프라인은 코드를 짜는 것보다 품질 기준을 세우는 과정이 더 어렵다. AgentCrow 패턴으로 평가-수정-재생성 루프를 병렬화했고, 반복 횟수를 줄이는 데 결정적이었다.

## "최악이야 결과물, 이전에 스킬 고도화한 거 어디 갔어?"

첫 번째 이미지 세트가 나왔을 때 피드백은 즉각적이었다. `naver-dental-blog` 스킬이 업데이트됐는데 새 기준이 파이프라인에 반영이 안 됐던 거다.

문제를 파악하고 에이전트 3개를 병렬로 투입했다.

```
🐦 AgentCrow — dispatching 3 agents:
1. @skill-updater → SKILL.md 사진 카테고리 7종 반영 + 사진 16장 정정
2. @blog-enhancer → 001 임플란트 블로그 글 전문성 고도화
3. @blog-reader → 002/003 블로그 구조 분석 (고도화 기준점)
```

AgentCrow는 `.claude/agents/` 디렉토리에 정의된 역할별 에이전트를 CLAUDE.md 규칙에 따라 디스패치하는 패턴이다. 메인 컨텍스트에서 직접 파일을 읽고 수정하는 대신, 에이전트가 스코프 내 작업을 완료하고 결과 요약만 올라온다. 이번 세션에서 Agent 도구가 23회 호출됐다.

## 로고 크기 3번, 배경색 4번, 한글 깨짐 5번

파이프라인이 돌아가기 시작한 뒤부터는 퀄리티 피드백 루프였다.

**로고 크기 조정 3라운드**
처음에는 너무 컸다 → "지금 사이즈의 반~1/3". 재생성했더니 너무 작아졌다 → "지금 사이즈의 2배". 스크린샷을 공유하면서 직접 비교했다. 최종적으로 `pipeline.py`의 로고 합성 파라미터를 3번 조정했다.

**배경색 4라운드**
`#ffffff` → 웜 베이지 `#f5f0eb` → 약한 하늘색 `#f0f4ff` → 다시 `#ffffff`. 색을 바꿀 때마다 HTML 템플릿 7종 전체를 에이전트가 수정하고 파이프라인을 재실행했다. 결론은 처음으로 돌아왔다. 프로세스 없이 방향을 정하지 않으면 반복이 늘어난다.

**한글 깨짐과 영어 혼입**

> "그림 무조건 한글 들어가게 해줘"

> "영어 없는 건 필수잖아"

Gemini가 생성하는 이미지에 영문 라벨이 섞여 나오는 문제가 반복됐다. `build_gemini_prompt()`에 명시적으로 한국어 전용 프롬프트 지시를 추가하고, 출력 이미지마다 한글 깨짐 여부를 검증하는 단계를 SKILL.md에 규칙으로 박았다.

## 파이프라인 실패가 background task로 숨어있었다

세션 중반부터 파이프라인 재실행이 계속 `exit code 2`로 실패했다.

```
Background command "캐시 삭제 + 파이프라인 재실행" failed (exit code 2)
Background command "파이프라인 재실행 (한글 라벨 전환)" failed (exit code 2)
Background command "캐시 삭제 + 파이프라인 재실행 (분할 이미지 포함)" failed (exit code 2)
```

Bash 백그라운드로 실행한 `nohup npx tsx` 스크립트들이 에러를 내는 게 `task-notification`으로 올라왔다. 포그라운드 실행이었다면 바로 보였을 에러들이다. 백그라운드 태스크를 쓸 때는 실패 감지 후 로그 확인 단계를 반드시 넣어야 한다는 걸 다시 확인했다.

## "과정을 한 장에 다 보여주는데 2장으로 나눠도 되지 않아?"

이미지 분할 요구사항이 들어왔을 때 구체적인 작업이 생겼다.

```
// Before: 6단계 과정을 2단계씩 묶어서 3장
보철방법-3가지비교.png   // 브릿지/틀니/임플란트 한 장에
임플란트-6단계.png       // 6단계를 한 장에

// After: 1단계 = 1장
보철방법-브릿지.png
보철방법-틀니.png
보철방법-임플란트.png
임플란트-1단계.png ~ 임플란트-6단계.png
```

`post.html`에서 IMAGE 태그를 분할하고, SKILL.md에 "과정 이미지는 단계당 1장" 규칙을 추가했다. 스킬에 규칙을 박아야 다음 글에서 처음부터 올바르게 나온다.

## 사진 16장 분류: manifest.json을 에이전트가 직접 만든다

`~/uddental/clinics/uddental/photos/`에 있는 KakaoTalk 사진 16장을 블로그에 쓰려면 카테고리 분류가 필요했다. 파일명이 `37-001.jpg`, `45-004.jpg` 식이라 내용을 봐야 안다.

에이전트가 16장을 전수조사하고 `manifest.json`을 생성했다.

```json
{
  "37-001": { "category": "exterior", "desc": "건물 정면 넓은 앵글" },
  "45-004": { "category": "ct-room", "desc": "CT/파노라마 장비 클로즈업" },
  "46-009": { "category": "waiting-room", "desc": "소파+TV 대기실" }
}
```

초기 분류에서 오류가 3건 있었다. `46-009`가 `진료실`로 분류됐는데 실제는 대기실이었고, `45-004`는 복도가 아니라 CT실이었다. 에이전트가 이미지를 직접 보고 분류해도 확인 단계는 필요하다.

## 3개 프로젝트, 454 tool calls

이번 기간 3개 세션의 통계다.

| 세션 | 프로젝트 | 시간 | tool calls |
|------|---------|------|-----------|
| 1 | spoonai | 13h 43min | 11 |
| 2 | uddental | 25h 5min | 261 |
| 3 | saju_global | 105h 43min | 182 |

도구별로는 Bash 174회, Read 138회, Agent 41회, Edit 35회, Grep 20회.

세션 1(spoonai)에서는 Cowork 스케줄이 2주째 죽어있는 걸 발견했다. `crontab -l` 비어있음, launchd 플리스트 없음. `durable: true` 옵션 없이 등록된 크론이 세션 만료와 함께 사라진 거다. 자동화 스케줄은 내구성(durability) 설정을 반드시 확인해야 한다.

세션 3(saju_global)에서는 SEO 궁합 랜딩 288페이지 구현을 `superpowers:writing-plans` 스킬로 플랜 작성 후 `subagent-driven-development`로 실행했다. 스펙 문서가 있으면 플랜 → 서브에이전트 디스패치 패턴이 가장 효율적이다.

이게 핵심이다. `pipeline.py`를 바꿔도 SKILL.md가 이전 규칙을 가리키고 있으면, 다음 세션에서 에이전트가 구버전 기준으로 작업한다. 코드와 스킬은 항상 동기화돼야 한다.
