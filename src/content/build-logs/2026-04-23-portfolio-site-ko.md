2026-04-22, 두 세션 합산 5시간 17분. Opus 4.7이 201번 도구를 호출했다. 코드는 한 줄도 안 썼다 — 전부 리서치 검증, 역공학, 프로토타이핑이었다.

**TL;DR** 5개 에이전트를 동시에 던져 66,745단어 dentalad 리서치를 도메인별로 검증했고, Claude Design 시스템 프롬프트 422라인을 분석해 로컬 `claude-design-lite` 스킬로 이식했다.

## 5개 에이전트, 하나의 문서를 동시에 뜯다

`dentalad/ads-research/`에 광고 리서치가 쌓여 있었다. V1 원본 12건 + V2 검증본 8건 + 통합 4건, 총 66,745단어. V2가 V1을 상당히 뒤집어놨는데 — 네이버 Cue:가 "핵심 알고리즘"으로 가정돼 있었으나 2026-04-09에 이미 종료됐고, 블로그 제작 원가는 2.6만원이 아니라 1,700원 + 인건비였다. 그 V2가 맞는지 다시 검증해야 했다.

도메인 5개로 쪼개서 에이전트를 동시에 날렸다:

```
1. 규제 → AI 기본법·공정위·대법원·의료법
2. 경쟁사 → 케어랩스 외 Top 5 + 글로벌
3. 플랫폼 → 네이버·Meta·ChatGPT 동향
4. 유닛 이코노믹스 → 원가·가격·MRR 구조
5. 시장 데이터 → CRM·ROAS·LTV·TAM 수치
```

순차로 돌렸으면 4~5시간짜리 작업이다. 병렬로 돌리니 단일 세션 안에 끝났다. 핵심은 각 에이전트에 "900단어 이내 팩트체크 리포트"라고 명시한 것 — 길면 컨텍스트가 터진다.

결과에서 나온 흥미로운 발견들이 있었다. 플랫폼 에이전트가 잡아낸 건 주어 오류였다. "ChatGPT가 네이버를 차단"이라는 문장이 원문에 있었는데, 실제로는 **네이버가 ChatGPT 봇을 차단**한 것이다. 방향이 반대였다. 규제 에이전트에서는 공정위 가상인물 규제가 "시행"이 아니라 "행정예고 단계"였다는 걸 잡았고, 경쟁사 에이전트는 케어랩스가 2024-10부터 매각 진행 중이라는 사실을 추가로 발굴했다 — 기존 dossier에 없던 내용이었다.

> 검증 에이전트는 "이게 사실이다"를 찾는 게 아니라 "이게 틀릴 수 있는 이유"를 찾도록 프롬프트해야 한다.

## Claude Design 시스템 프롬프트 422라인 역공학

세션 2는 방향이 달랐다. 시작 프롬프트: "claude design 코드 유출된 거 찾아줘". `claude.ai/design`은 2026-04-17 출시된 Anthropic Labs 신제품이다.

웹 검색으로 `elder-plinius/CL4R1T4S` 레포에서 `ANTHROPIC/Claude-Design-Sys-Prompt.txt`를 찾았다. 422라인, 약 73KB. 커밋 이력이 출시일(2026-04-17)과 정확히 일치해서 최신본이 맞다고 판단했다.

실제 소스 코드 유출은 없었다. Claude Code TypeScript 51만 3천 줄은 2026-03-31 npm sourcemap 사고로 나왔지만, Claude Design의 React 코드는 공개되지 않았다. 유출된 건 시스템 프롬프트와 툴 스키마뿐.

그 422라인으로 내부 구조를 역공학했다:

```
역할: "HTML을 도구로 쓰는 전문 디자이너"
출력: HTML이 유일한 네이티브 포맷
기반 모델: Claude Opus 4.7 (고정)
환경: 파일시스템 기반 프로젝트 워크스페이스
```

일반 Claude 채팅과 다른 점은 **파일시스템 기반 프로젝트**라는 것이다. 상대 경로로 파일을 참조하고 프로젝트 간 격리가 된다. 그리고 이건 프롬프트나 스킬이 아니라 `claude.ai/design`에 접속하는 별도 웹 제품이다 — Pro/Max/Team/Enterprise 전용이고 무료 플랜은 불가.

이걸 CLI에서 재현 가능한 부분만 뽑아 `~/.claude/skills/claude-design-lite/SKILL.md`를 만들었다. Live Preview, Tweaks 패널 같은 호스트 의존 기능은 버리고 **질문 기법·컨텍스트 수집·변주 생성·AI-slop 가드**만 이식했다. Claude Design이 사용자에게 던지는 10가지 질문 패턴을 분석해서 스킬 안에 박아 넣었다.

## jidonglab.com 리디자인, 4개 방향 동시에

스킬을 만들자마자 바로 실전 투입했다. `~/jidonglab-redesign/`에 방향 4개를 동시에 생성했다:

- `v1-notebook.html` — 노트북 스타일
- `v2-pro.html` — 테크 포트폴리오
- `v2-studio.html` — 스튜디오 느낌
- `v3-labos.html` — 실험적 방향

피드백은 "2번 괜찮은데 좀 더 고도화할 거 없어? 테크 포트폴리오 느낌으로". 그 다음에 나온 요구가 실제 커밋·포스트·빌드로그 데이터를 하나의 히트맵으로 보고 싶다는 것이었다. 카피 방향도 확정됐다:

> "매일, 기록한다. 1년치 커밋·포스트·빌드로그를 한 화면에. 놀았던 날도, 폭주한 날도 숨기지 않는다."
