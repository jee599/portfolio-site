번역 버튼이 안 된다는 건 알고 있었다. 정확히는 "잘 안 된다"는 표현이 맞다. 눌리는 것 같은데 실제로 언어가 제대로 안 바뀌는 느낌. 원인을 파악하고 배포까지 24분, 50 tool calls로 끝냈다.

**TL;DR** `var lang = 'ko'` 한 줄이 전체 번역 시스템을 망가뜨리고 있었다. 기본값을 영어로 바꾸고 버튼 로직을 뒤집는 작업에 Opus 4.6이 50번의 도구 호출을 썼다.

## "번역 버튼이 잘 안 돼요"

요청은 단순했다. "영어 사이트가 기본이어야 하고, 영어 번역이 모든 부분에 되어 있어야 해. 영어 기본으로 하고 한국어 번역 버튼 넣어줘."

Opus 4.6이 먼저 현재 번역 시스템 상태를 파악했다. `Base.astro`를 읽으니 핵심 문제 3개가 나왔다.

첫째, `var lang = 'ko'`로 기본 언어가 하드코딩되어 있었다. 둘째, 버튼 텍스트가 반대였다. 영어가 기본이면 "KO" 버튼이 보여야 하는데 "EN"이 보이고 있었다. 셋째, 126개 빌드 로그에 영어 번역이 없었다. 이건 별도 작업으로 남겼다.

원인은 명확했다. localStorage에서 `lang` 키가 없을 때 fallback이 `'ko'`였다. 사이트를 처음 방문하는 영어권 사용자는 항상 한국어 화면을 봤다.

## data-ko/data-en 속성 시스템

이 포트폴리오 사이트의 i18n 방식은 특이하다. 서버사이드 번역이 아니라, 모든 텍스트 요소에 `data-ko`와 `data-en` 속성을 달아두고 JavaScript가 현재 언어에 맞는 속성 값으로 `textContent`를 교체하는 방식이다.

```html
<span data-ko="최근 포스트" data-en="Recent Posts">Recent Posts</span>
```

기본 텍스트가 영어로 되어 있어야 JavaScript 실행 전에도 영어가 보인다. 그런데 기존 코드는 기본 텍스트 자체가 한국어였다. 페이지 로드 시 잠깐 한국어가 보이다가 JavaScript가 실행되면서 영어로 바뀌는 깜빡임도 있었다.

수정은 3단계였다. `<html lang="ko">` → `<html lang="en">`, `var lang = 'ko'` → `var lang = 'en'`, 버튼 기본 텍스트 `EN` → `KO`. 그리고 모든 `data-ko`/`data-en` 요소의 기본 텍스트를 영어로 변경했다. `Base.astro`, `PostLayout.astro`, `blog/[slug].astro` 세 파일을 건드렸다.

## 50 tool calls가 필요했던 이유

Read 13번, Bash 12번, Edit 12번, Agent 8번, Glob 2번.

에이전트 8번이 눈에 띈다. 번역 검수를 병렬 에이전트에 위임했다. "모든 영어 번역 잘 되는지, 번역 매끄러운지 오타 없는지 찾아봐"라는 요청에 AgentCrow가 작동했다. 단일 에이전트가 전체 사이트를 스캔하는 대신 섹션별로 에이전트를 분산해서 검수했다.

Bash 12번은 대부분 빌드 확인과 배포 과정이었다. Astro 사이트라 `astro build` 실행 후 에러 없음을 확인하고, 그다음 Cloudflare Pages 배포.

나머지 Read/Edit은 세 파일을 반복적으로 읽고 수정하는 과정이었다. 변경할 부분이 각 파일에 흩어져 있어서 한 번에 끝나지 않았다.

## 삽질: 126개 빌드 로그

실제로 번거로운 건 기존 126개 빌드 로그 파일이었다. 이 파일들은 `lang: ko`로 작성된 한국어 포스트다. 영어 페어(`lang: en`)가 없으면 빌드 로그 목록에서 영어 사용자는 한국어 제목만 본다.

이번 세션에서는 해결하지 않았다. 요청이 "번역 버튼 고쳐줘"였지 "기존 콘텐츠 전부 번역해줘"가 아니었으니까. Opus 4.6도 스코프를 정확히 인식하고 UI 레이어 문제만 해결했다.

기존 포스트 영어 번역은 별도 작업으로 남겨뒀다. 126개를 한 번에 번역하는 건 Claude API로 자동화할 수 있는 작업이라 나중에 스크립트로 처리할 예정이다.

## Before / After

| 항목 | Before | After |
|------|--------|-------|
| 기본 언어 | 한국어 (hardcoded) | 영어 |
| 버튼 텍스트 | "EN" | "KO" |
| `<html lang>` | `ko` | `en` |
| 첫 방문 경험 | 한국어 화면 | 영어 화면 |
| 로드 깜빡임 | 있음 (ko→en 전환) | 없음 |

배포 후 사이트를 열었을 때 기본 언어가 영어로 나오고, 우측 상단에 "KO" 버튼이 보이면 성공. 버튼 클릭 시 한국어로 전환, 다시 클릭 시 영어로 돌아오는 토글도 정상 동작했다.

24분, 50 tool calls. 번역 버튼 하나 고치는 데 50번이 필요했다는 게 처음엔 많아 보였는데, 에이전트 8번 포함한 숫자라 체감 작업량보다 수치가 크다. 에이전트 병렬 검수가 없었으면 Bash로 직접 파일 하나씩 확인하는 데 더 많은 시간이 걸렸을 것이다.
