## GLM-5 — NVIDIA 없이 훈련한 오픈소스 1위 모델

중국 Zhipu AI가 GLM-5를 MIT 라이센스로 공개했다. 744B 파라미터 중 40B만 활성화되는 MoE(Mixture of Experts) 구조로, SWE-bench Verified 77.8%를 기록하며 오픈소스 모델 중 1위에 올랐다.

특히 주목할 점은 훈련 인프라다. NVIDIA GPU 없이 Huawei Ascend 칩만으로 훈련을 완료했다. 미국의 대중국 AI 칩 수출 규제가 중국 AI 생태계를 위축시키기는커녕, 대안 하드웨어 기반의 독자적인 기술 스택을 가속화하고 있다는 증거다.

API 가격도 경쟁력이 있다. Input $1.00/1M tokens, Output $3.20/1M tokens로, frontier 모델치고는 합리적인 수준이다.

MIT 라이센스는 상업적 사용에 제한이 없다는 뜻이다. 기업들이 자체 인프라에 배포하거나, fine-tuning해서 제품에 통합하는 것이 자유롭다. Meta의 Llama가 열었던 오픈소스 LLM 시장에 강력한 경쟁자가 추가됐다.

## Microsoft Phi-4-Reasoning-Vision — 작지만 강한 15B 모델

Microsoft가 Phi-4-reasoning-vision-15B를 오픈소스로 공개했다. 15B 파라미터로 훨씬 큰 시스템들과 동등하거나 우수한 성능을 보여준다.

이 모델의 독특한 점은 두 가지다. 첫째, **멀티모달 reasoning**이 가능하다. 이미지를 보고 논리적으로 추론하는 능력을 갖추고 있다. 둘째, **adaptive chain-of-thought**를 도입했다. 간단한 질문에는 바로 답하고, 복잡한 문제에만 step-by-step 추론을 자동으로 활성화한다. 불필요한 추론 오버헤드를 줄이면서도 필요할 때는 깊은 사고를 할 수 있다.

훈련 효율성도 인상적이다. 240대의 NVIDIA B200 GPU로 단 4일 만에 훈련을 완료했다. 수천 대의 GPU와 수 주의 훈련이 필요한 대형 모델과 비교하면, 자원 효율성에서 압도적이다.

15B 규모는 consumer GPU에서도 돌릴 수 있는 크기다. 실질적으로 개인 개발자나 소규모 팀이 로컬에서 활용 가능한 성능 상한선을 끌어올린 셈이다.

## Alibaba Qwen3.5-397B-A17B — MoE 대형 모델의 진화

Alibaba가 Qwen3.5-397B-A17B를 공개했다. 397B 파라미터 중 17B만 활성화되는 대규모 MoE 아키텍처다.

멀티모달 추론과 초장문 컨텍스트를 모두 지원한다. 이전 세대 대비 디코딩 처리량이 8.6x에서 최대 19x까지 향상됐다. 같은 하드웨어에서 훨씬 더 빠르게 응답할 수 있다는 의미다.

MoE 구조의 장점이 점점 더 분명해지고 있다. 전체 파라미터가 크더라도 실제로 활성화되는 파라미터는 적기 때문에, 추론 시 연산 비용을 크게 줄일 수 있다. GLM-5(744B/40B), Qwen3.5(397B/17B) 모두 이 전략을 채택하고 있다. "큰 모델을 만들되, 효율적으로 돌리는" 방향이 오픈소스 LLM의 주류 트렌드가 되고 있다.

## MCP — AI의 USB-C가 되다

Anthropic이 만든 Model Context Protocol(MCP)이 사실상의 업계 표준으로 자리잡았다. OpenAI, Microsoft, Google이 모두 채택했다.

MCP는 AI 모델이 외부 도구, 데이터 소스, API와 상호작용하는 방식을 표준화하는 프로토콜이다. AI 모델마다 도구 연결 방식이 달랐던 파편화 문제를 해결한다. "USB-C for AI"라는 별명이 붙은 이유다.

개발자 입장에서 MCP의 가치는 명확하다. 한 번 MCP 서버를 구현하면, Claude든 GPT든 Gemini든 어떤 모델에서도 같은 도구를 사용할 수 있다. 모델을 바꿀 때마다 도구 통합 코드를 다시 작성할 필요가 없어진다.

경쟁사의 프로토콜이 업계 표준이 되는 것을 Google과 OpenAI가 수용한 것은, 그만큼 표준화의 필요성이 절실했다는 방증이다. AI 에이전트 시대가 본격화되면, MCP의 역할은 더 커질 수밖에 없다.

## Yann LeCun, Meta를 떠나 World Model 연구소 설립

AI 분야의 거장 Yann LeCun이 Meta를 떠났다. 독립 연구소를 설립하며 $5B 밸류에이션을 목표로 한다.

LeCun이 집중하려는 분야는 "world model"이다. 3D 공간에서 사물이 어떻게 이동하고 상호작용하는지를 학습하는 모델로, 현재의 LLM과는 근본적으로 다른 접근이다. LeCun은 오랫동안 "LLM만으로는 진정한 AI에 도달할 수 없다"고 주장해 왔고, 이제 그 비전을 직접 실행에 옮기는 것이다.

이 움직임은 AI 연구 생태계에 영향을 줄 수 있다. LeCun급 연구자가 대기업에서 독립하는 것은, "빅테크의 AI 연구 독점"에 대한 반작용의 신호탄이 될 수 있다. 연구의 방향성을 기업의 제품 로드맵이 아닌, 순수한 과학적 호기심이 결정하는 환경을 만들겠다는 것이다.

## GitHub 트렌딩 — AI/LLM 관련 주목 프로젝트

이번 주 GitHub에서 활발히 업데이트되고 별을 받은 AI/LLM 관련 프로젝트를 정리한다.

**[ollama/ollama](https://github.com/ollama/ollama)** — 164K+ stars
LLM을 로컬에서 실행하는 가장 쉬운 방법이다. Kimi-K2.5, GLM-5, MiniMax, DeepSeek, Qwen, Gemma 등 최신 모델을 `ollama run` 한 줄로 돌릴 수 있다. 새로운 오픈소스 모델이 나올 때마다 빠르게 지원을 추가하고 있다.

**[langgenius/dify](https://github.com/langgenius/dify)** — 131K+ stars
Agentic workflow 개발을 위한 프로덕션 레디 플랫폼이다. 시각적 워크플로우 빌더, RAG 파이프라인, 에이전트 프레임워크를 제공한다. LangChain보다 더 완성도 높은 올인원 솔루션을 지향한다.

**[firecrawl/firecrawl](https://github.com/firecrawl/firecrawl)** — 89K+ stars
웹사이트 전체를 LLM-ready markdown이나 structured data로 변환하는 Web Data API다. RAG 파이프라인에 웹 데이터를 넣을 때 유용하다. 크롤링, 파싱, 정제를 한 번에 처리한다.

**[vllm-project/vllm](https://github.com/vllm-project/vllm)** — 72K+ stars
고처리량, 메모리 효율적인 LLM 추론 및 서빙 엔진이다. PagedAttention 기술로 GPU 메모리를 효율적으로 관리한다. 프로덕션 환경에서 LLM을 서빙할 때 사실상 표준으로 자리잡았다.

**[OpenHands/OpenHands](https://github.com/OpenHands/OpenHands)** — 68K+ stars
AI 기반 소프트웨어 개발 플랫폼이다. AI 에이전트가 코드를 작성하고, 테스트하고, 디버깅하는 전체 개발 사이클을 자동화한다.
